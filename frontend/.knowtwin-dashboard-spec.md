## Producto: qué es KnowTwin

Plataforma de captura de conocimiento tácito para offboarding empresarial.
Fork de EcoDB. Cliente: Manu. Backend: 120+ endpoints, FastAPI, PG+AGE+pgvector.

**Flujo del curator:**
1. Upload docs → worker chunking → entity extraction (GLiNER)
2. Curator batch: extrae claims de docs, promue, detecta contradicciones
3. Interview: empleado valida/contradice claims en tiempo real (WS)
4. Dispute resolution: claims conflictivos se resuelven side-by-side
5. Twin query: consulta semántica del conocimiento capturado
6. Coverage monitoring: % de conocimiento capturado por entidad

**Dato core:** el CLAIM (no "memory"). 5 ejes de estado:
- corroboration_level: draft → single_source → corroborated → corroborated_by_employee → validated | rejected
- dispute_state: undisputed → disputed → resolved_in_favor | resolved_against
- freshness_state: active → stale → dormant
- trust_tier: 0-2
- confidence: 0-1

**Demo seed:** Juan Garcia, 34 claims (24 doc + 10 interview), 16 disputed, 8 docs, 60 entities, coverage 9%, Banco Norte 46.3%.
